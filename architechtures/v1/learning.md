# Learning Log

- For the text-JEPA tokenizer/masking milestone, treat `Qwen/Qwen3-0.6B` as a Hugging Face `AutoTokenizer` source, but do not assume it has a native JEPA-ready mask token; the wrapper should verify `mask_token` and add one explicitly if absent.
- Do not reuse Hugging Face `DataCollatorForWholeWordMask` for the Qwen path; it is BERT-oriented, so span/block masking should stay in a custom tokenizer-aware module.
- The tokenizer/masker path depends on fast-tokenizer `offset_mapping`; fail early if the loaded Hugging Face tokenizer is not fast.
- Keep masker unit tests offline and deterministic with a tiny fake tokenizer; reserve real Hugging Face loading for the tokenizer wrapper boundary.
- Layer is the codename for the text-first JEPA latent-prediction system in this workspace; use the JEPA paper as the core architectural reference and treat LLM-JEPA as adjacent precedent rather than the baseline design.
- Keep EMA compact as a single `update_ema(target_module, source_module, momentum)` function; `momentum=0.0` acts as the initialization copy and ongoing updates should read `model.ema_momentum` from YAML.
- For tiny FineWeb experiments, use a local JSONL sample and filter by `token_count` / `language_score` before training; the dataset path should mask deterministically per example via `Random(seed + index)` so dataloader shuffling does not change the masking contract.
- Keep the Layer batch boundary aligned with the collated dataset output: `LayerModel.forward` should accept the full batch dict shape, including currently unused `target_mask` and `target_token_ids`, so `model(**batch)` works in the trainer without special filtering.
- Keep checkpointing in `scripts/train.py` as small script-level helpers; save step-specific files plus `latest.pt`, and note that tokenizer startup can still fail independently on Hugging Face network timeouts before any checkpoint code runs.
- The real LLM-JEPA path is a separate paired-view causal-LM setup: use `messages`/`text`/`code` JSONL examples, append predictor special tokens to the source view, compare source and target hidden states with a JEPA loss, and keep the standard LM loss in the same objective instead of reusing the Layer EMA masked-span stack.
- Keep the LLM-JEPA dataset contract explicit: require `text` and `code` views in each row, append predictor tokens to the last source-view message, and reject source/target JEPA views that exceed `max_length` rather than truncating away the intended readout semantics.
- For existing LLM-JEPA datasets that only ship `messages`, allow them only when they are unambiguous single-turn prompt->assistant rows; derive the JEPA source from the full non-assistant prefix and reject multi-turn assistant histories unless explicit `text`/`code` views are provided.
- In chat-template tokenizers, JEPA readout indices cannot be chosen as the last visible token because templates may end with special markers plus plain newline tokens; source readout should target the final predictor token explicitly, and target readout should scan backward to the last non-special, non-whitespace token.
- `sshleifer/tiny-gpt2` is a bad cosine-JEPA smoke model because its hidden size is 2; after LayerNorm, 2D hidden states collapse onto a single line, so cosine similarities saturate at `+/-1` and JEPA loss can be spuriously near `0` or `2` independent of nuanced semantic alignment.
- On Apple Silicon in this repo, prefer `mps` over `cpu` for local trainer defaults; `cuda`-only fallback logic makes LLM-JEPA look hung because a single Qwen step on CPU is extremely slow.
- For large LLM-JEPA checkpoints on a low-free-space machine, do not duplicate the payload as both `step-*.pt` and a second physical `latest.pt`; hard-link `latest.pt` to the step checkpoint, and keep the local LLM-JEPA checkpoint payload model-only to stay within disk limits.
- Keep benchmark task logic centralized under `src/text_jepa/benchmarking.py`; both local and OpenRouter benchmark scripts should share the same dataset prompt shaping and exact-score rules, including Spider SQL execution and GSM8K final-answer extraction.
- Keep runtime reproducibility centralized under `src/text_jepa/utils/repro.py`; trainer scripts should read `runtime.seed` and `runtime.deterministic` from YAML by default, allow CLI overrides, and use one shared helper to seed Python/PyTorch and enable deterministic algorithms.
- OpenRouter reasoning-capable models can return `content: null` and spend the whole completion budget on reasoning unless benchmarking requests explicitly disable reasoning; in `scripts/benchmark_openrouter.py`, keep `reasoning={"effort": "none", "exclude": True}` and use the documented `max_tokens` field for chat completions.
- Local LLM-JEPA checkpoint loading must rebuild the tokenizer from the training config path stored in the checkpoint, not from the backbone model name alone; this repo can train a small backbone against the YAML tokenizer (for example Qwen tokenizer + tiny GPT-2 backbone), and loading with the wrong tokenizer will fail on embedding-size mismatches.
- Local `Qwen/Qwen3-*` benchmarking should prepend `/no_think` and strip any residual `<think>...</think>` blocks from generations before scoring; otherwise the model may emit reasoning text instead of just the answer.
- On this Apple Silicon machine, local Qwen LLM-JEPA training on `mps` can stall after initialization, while the same one-step run completes on `cpu` given enough time; use CPU as the fallback path for real checkpoint creation when MPS shows no step progress.
- Local benchmarking now supports optional hidden-state similarity analysis in `scripts/benchmark_local.py`; keep it local-only because OpenRouter benchmarking cannot expose embeddings, and summarize cosine similarity separately for matched vs unmatched examples to mirror the old `llm-jepa/evaluate.py` tune/untune split.
- Intertwined H-JEPA is a separate predictive residual-stream architecture, not a standard LayerModel variant: each non-final layer predicts a delta in compressed space toward an EMA copy of the next CE path (`ce_norm + compressor`) applied to the next post-attention residual state, and the predictor delta remains active at inference.
- Literature pass for Intertwined H-JEPA: keep borrowed tokenizer path + LM head because LLM-JEPA, data2vec, and NextLat support language latent prediction with token objectives, but treat residual-delta injection as part of the architecture in the first pass.
- Refined Intertwined H-JEPA training contract: `h_l_normed = LayerNorm(h_l)`, `h_l_post_attn = h_l + Attention_l(h_l_normed)`, `z_l = CE_l(LayerNorm(h_l_post_attn))`, `delta_l = Pred_l(z_l)`, `h_{l+1} = h_l_post_attn + Proj_l(z_l + delta_l)`.
- JEPA target contract: `target_z_l = CEbar_{l+1}(LayerNorm(h_{l+1}_post_attn))` under `no_grad`, and `loss_jepa_l = MSE(delta_l, target_z_l.detach() - z_l.detach())`.
- First-pass Intertwined H-JEPA should not include SIGReg, VICReg, cosine loss, or normalized-MSE tricks. Start with plain MSE JEPA loss and next-token LM loss.
- Error handling policy for first-pass Intertwined H-JEPA: avoid broad defensive checks; manually check only confusing architecture invariants like `depth >= 2`, JEPA loss shape agreement, and nonempty masked loss selections.
- EMA policy for first-pass Intertwined H-JEPA: EMA tracks the CE path (`ce_norm + compressor`) only, excludes predictors/projectors/attention/embeddings/LM head, is hard-copied at init, is not in AdamW, computes targets under `no_grad`, and updates only after `optimizer.step()`.
- Use `make_ema_copy(module)` for EMA setup: `copy.deepcopy(module)`, then `ema.requires_grad_(False)`, then `ema.eval()`. This is cleaner than looping over each EMA parameter.
- First-pass JEPA loss: use normal MSE only, `loss_jepa_l = MSE(delta_l, target_z_l.detach() - z_l.detach())`; start with `lambda_jepa = 0.1`.
- Weight tying policy: optionally set `lm_head.weight = token_embedding.weight` only when both are `(vocab_size, residual_dim)`; do not add projection glue just to force tying, and keep embeddings out of EMA either way.
- Autoregressive usage clarification: one inference pass through the JEPA stack produces logits for one next token, then that token is appended and the model is run again. Training may still use teacher forcing over full sequences for efficiency.
- First-pass embedding/LM decisions: use plain token + position embeddings, every block has CE/Pred/Proj, call the EMA target `target_z` instead of `zstar`, and train the LM head with next-token prediction rather than same-token reconstruction.
- Intertwined H-JEPA should borrow Nanochat's small-script trainer/checkpoint discipline, but not its scale-driven systems stack in v1: start with AdamW, simple token chunking, naive generation, and explicit diagnostics before considering Muon, FP8, distributed loading, packing, or KV-cache machinery.
- Intertwined H-JEPA v1 training should integrate `wandb` from the first real trainer and keep a chat-shaped data/rendering path available from the start: support conversation JSONL plus explicit role markers so training, sampling, and later chat-style evaluation share one tokenizer/rendering contract instead of separate raw-text and chat pipelines.
- Intertwined H-JEPA trainer now treats JEPA supervision as explicit per-layer local losses on all non-final layers, sums them into one scalar JEPA term scaled by YAML `lambda_jepa`/warmup, combines that with the global LM loss, and calls a single `backward()` on the total objective after which EMA updates run post-optimizer-step.
- The current trainer uses PyTorch `DataLoader` helpers for both the infinite pre-batched FineWeb training stream and reshaped validation windows, times steps with explicit device synchronization around the whole step, and clears gradients with `model.zero_grad(set_to_none=True)` only after `optimizer.step()`/`update_ema()`.
- SIGReg is implemented as a local sliced Epps-Pulley objective matching `lejepa`: random normalized 1D projections, `[0, t_max]` characteristic-function quadrature, and mean slice statistic. In Intertwined H-JEPA it is applied to non-final layer representations without detaching `h_l_post_attn`, so SIGReg gradients can flow through the representation path and earlier computation while still avoiding EMA targets.
- On the small MPS smoke with `B=2`, `L=64`, `num_slices=16`, raw summed SIGReg across three non-final layers was about `138-140`; start with very small `beta_sigreg` values such as `0.001` when enabling it.
- `scripts/inspect_checkpoint.py` is the lightweight checkpoint sanity tool: it loads checkpoint config as truth, resolves dataset/tokenizer through `parameter-golf`, asserts generic vocab agreement, reports eval loss contributions, per-layer `z`/delta diagnostics, and short generations. First run on the 200-step pre-SIGReg checkpoint showed high generation repetition and much lower `z_std_mean` in layers 1-2 than layer 0.
